What Hardening Covers — 3 Areas*

1 — Framework currency.NET Core 2.2 (unsupported since 2019) upgraded to current LTS. Angular 1 (unsupported since Jan 2022) dependency risk addressed. The 9 known P0 bugs fixed, including the Stripe ID-mismatch bug that can block all platform submissions.
2 — Email & billing integrationSelf-service SendGrid domain setup (DKIM/DMARC/DNS) replacing the manual, multi-day process that currently derails new client onboarding. Stripe remediation restores the ability to create products and adjust pricing.
3 — Monitoring baselineBasic error logging and monitoring so failures are caught before clients report them. Platinum tier extends this to formal backup/DR and audit logging.

What This Path Requires You to Accept

Hardening is not a build. Going in with eyes open means accepting the following:

No new capabilityLegacy Fund applications, SMART IR automation, and portfolio dashboards stay fully manual through and after this work. Nothing here increases what the platform can do — only how reliably it does what it already does.
Tier durabilitySilver-tier fixes (patches, dependency bumps) are largely non-transferable if a full rebuild happens later. Confirmed for Stripe specifically by D-012: production billing is architected around Stripe Elements, Customer Portal, and webhook sync — a wholesale replacement, not an extension of today's integration. The Stripe fixes in this path carry ~zero value forward into a rebuild.
Shared budgetThis ~$100k competes inside the same $500k pool as salaries, marketing, and debt payoff — it is not incremental to that total. Per the July 16 math, the full $500k is projected to be spent within three months across all categories combined.
If demand acceleratesHardening targets reliability for 50–100 clients, not unlimited scale. If Legacy Fund launches drive signups well beyond that, Path 1 or Path 2 investment is still required — hardening alone does not remove that need, it only buys time to fund it properly.
Important: Path 3 is not an alternative to Path 1 or Path 2 — it's what makes either possible without the current platform failing first. The existing AI-generated interactive mockup (demo-only, no backend) is a separate asset for fundraising conversations; it is neither Path 2's working pilot nor Path 3's hardened production system.
